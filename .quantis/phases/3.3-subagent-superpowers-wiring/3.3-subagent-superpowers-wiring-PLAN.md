# Phase 3.3 · Subagent & Workflow Skill-Wiring — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Read and follow `.agents/skills/subagent-driven-development/SKILL.md`. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Make every dispatched subagent actually use the superpowers methodology it embodies, by handing it the skill **path** (read-on-demand) at the dispatch site — never pasted skill text. Plus reconcile the one routing contradiction and the two carried-in workflow-delegation gaps.

**The contract (applies to every task):** a subagent dispatch carries `REQUIRED SUB-SKILL: Read and follow .agents/skills/<X>/SKILL.md` for the methodology that subagent embodies. Lean prompt + path = no `self`-stall (D-009) AND real superpowers (D-010).

**Architecture:** One task per file. Tasks execute top-to-bottom (the contract lands first, the validator last). Prose/process edits only — no product code.
