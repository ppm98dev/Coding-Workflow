# Phase 2.1: Dynamic Skill and MCP Context Injection - Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use subagent-driven-development (recommended) or executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement dynamic custom skill and MCP discovery and context injection inside the Quantis planning workflows to leverage third-party ecosystem tools (e.g. from skills.sh and active MCP servers) without breaking core skill upgradability.

**Architecture:** We will modify `.agent/workflows/plan.md` to scan the `.agents/skills/` directory for custom folders (excluding core manifest folders) and read available lazy/eager MCP tools registered in the current session environment. We will display a beautiful discovery card to the planning agent to dynamically inject this context.

**Tech Stack:** Bash, Markdown, Git

### Task 1: Implement Dynamic Custom Skill Discovery

**Files:**
- Modify: `.agent/workflows/plan.md`

- [ ] **Step 1: Write the Custom Skill Discovery Bash Code**
  We want to scan `.agents/skills/` for custom skills, excluding the 18 core manifest skills listed in `MANIFEST.md`.
  Write and test this bash block:
  ```bash
  # Discover custom skills
  echo "💡 DISCOVERED CUSTOM SKILLS (from skills.sh):"
  CUSTOM_SKILLS_FOUND=0
  for skill_dir in $(ls .agents/skills/ 2>/dev/null); do
      # Cross-reference with MANIFEST.md to exclude core skills
      if ! grep -q "^- $skill_dir$" MANIFEST.md; then
          # Found a custom skill! Read the description from its SKILL.md frontmatter
          SKILL_FILE=".agents/skills/$skill_dir/SKILL.md"
          if [ -f "$SKILL_FILE" ]; then
              DESC=$(grep -A5 "^description:" "$SKILL_FILE" 2>/dev/null | head -n 1 | sed -E 's/description:? (.*)/\1/' | tr -d '"' | tr -d "'")
              echo "  - $skill_dir: ${DESC:-No description provided}"
              CUSTOM_SKILLS_FOUND=$((CUSTOM_SKILLS_FOUND + 1))
          fi
      fi
  done
  if [ "$CUSTOM_SKILLS_FOUND" -eq 0 ]; then
      echo "  - None installed (visit https://www.skills.sh to install custom skills)"
  fi
  ```

- [ ] **Step 2: Dry-run check**
  Run the bash snippet inside the repository terminal to verify that it executes cleanly on macOS/Linux and outputs a "None installed" or lists any active custom folders correctly.
  
- [ ] **Step 3: Commit**
  ```bash
  git add .agent/workflows/plan.md
  git commit -m "feat(phase-2.1): draft custom skill discovery logic in plan workflow"
  ```

### Task 2: Implement Active MCP Server Discovery

**Files:**
- Modify: `.agent/workflows/plan.md`

- [ ] **Step 1: Write the MCP Server Discovery Bash Code**
  Inspect `/Users/PietroPaolo/.gemini/antigravity-ide/mcp/` (using the environment path `$HOME/.gemini/antigravity-ide/mcp/`) to identify eager or lazy loaded MCP servers and extract their tool names from JSON schema filenames.
  Write and test this bash block:
  ```bash
  # Discover connected MCP servers
  echo "🔌 CONNECTED MCP SERVERS:"
  MCP_DIR="$HOME/.gemini/antigravity-ide/mcp"
  MCP_FOUND=0
  if [ -d "$MCP_DIR" ]; then
      for server in $(ls "$MCP_DIR" 2>/dev/null); do
          if [ -d "$MCP_DIR/$server" ]; then
              # List tools by parsing the .json schemas
              TOOLS=$(ls "$MCP_DIR/$server" 2>/dev/null | grep "\.json$" | sed 's/\.json//g' | tr '\n' ' ' | sed 's/ $//')
              echo "  - $server: tools -> [ $TOOLS ]"
              MCP_FOUND=$((MCP_FOUND + 1))
          fi
      done
  fi
  if [ "$MCP_FOUND" -eq 0 ]; then
      echo "  - None connected"
  fi
  ```
