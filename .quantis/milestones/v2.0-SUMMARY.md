## Lessons Learned
- **Dogfooding works**: Using GSD to build GSD surfaced real workflow issues (dead code from removals, context bloat from PowerShell blocks)
- **Removing > Reordering**: Removing PowerShell entirely was far more valuable than reordering Bash/PowerShell blocks. The 10% context savings compound across every session.
- **Framework is well-designed**: The core SPEC → PLAN → EXECUTE → VERIFY pipeline is solid. The "for Antigravity" adaptation was the weak link, not the methodology.
- **Key gap identified**: GSD outputs "it works" code, not production code. No error handling, logging, validation, or security defaults. This is the #1 priority for v2.1.
