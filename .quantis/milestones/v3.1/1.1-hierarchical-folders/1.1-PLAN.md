# Phase 1.1: Hierarchical Subphase Folder Architecture - Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use subagent-driven-development (recommended) or executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate Quantis' phase directory logic to support descriptive hierarchical folders (e.g. `.quantis/phases/1.1-hierarchical-folders/`) and update all workflows and skills to resolve paths dynamically.

**Architecture:** We will update all workflow bash scripts to resolve the active phase directory using prefix-based glob matching (e.g., finding the directory starting with the prefix `N.M-` in `.quantis/phases/`). We will also update all markdown skills to document and direct the creation of these new paths.

**Tech Stack:** Bash, Markdown, Git

### Task 2: Migrate Core Workflows

**Files:**
- Modify: `.agent/workflows/plan.md`
- Modify: `.agent/workflows/execute.md`
- Modify: `.agent/workflows/verify.md`

- [ ] **Step 1: Update `.agent/workflows/plan.md`**
  Modify the Delegate to Skill section. Add bash logic to look up the phase title from `ROADMAP.md`, normalize it into a slug, and construct the path:
  ```bash
  # Lookup Phase name from ROADMAP.md
  PHASE_TITLE=$(grep -i "Phase $PHASE" .quantis/ROADMAP.md | head -n 1 | sed -E 's/.*Phase [0-9.]+:? (.*)/\1/' | tr -d '\r')
  # Normalize to kebab-case slug
  PHASE_SLUG=$(echo "$PHASE_TITLE" | tr '[:upper:]' '[:lower:]' | sed -E 's/[^a-z0-9]+/-/g' | sed -E 's/^-+|-+$//g')
  PHASE_DIR=".quantis/phases/${PHASE}-${PHASE_SLUG}"
  ```
  Ensure the `writing-plans` delegation instructions direct saving the plan into this dynamic directory.

- [ ] **Step 2: Update `.agent/workflows/execute.md`**
  Modify the execution logic to find the phase directory dynamically:
  ```bash
  PHASE_DIR=$(find .quantis/phases -maxdepth 1 -name "${PHASE}-*" | head -n 1)
  if [ -z "$PHASE_DIR" ]; then
      echo "Error: phase directory starting with ${PHASE}- not found"
      exit 1
  fi
  # Use $PHASE_DIR instead of .quantis/phases/$PHASE/
  ```
  Update plan lookup to search `$PHASE_DIR` for `${PHASE}-PLAN.md` (or dynamic plan files).

- [ ] **Step 3: Update `.agent/workflows/verify.md`**
  Update verification paths to resolve the folder dynamically:
  ```bash
  PHASE_DIR=$(find .quantis/phases -maxdepth 1 -name "${PHASE}-*" | head -n 1)
  if [ -z "$PHASE_DIR" ]; then
      echo "Error: phase directory starting with ${PHASE}- not found"
      exit 1
  fi
  ```
  Update all reading of plans and writing of `VERIFICATION.md` and `SUMMARY.md` files to target `$PHASE_DIR/`.

- [ ] **Step 4: Verify and Commit**
  Run `git diff` to verify the core workflow changes.
