## Lessons Learned
1. **Fork vs reference**: Forking skills gives us freedom to rebrand and adapt, but creates maintenance burden. MANIFEST.md helps track what's ours vs upstream.
2. **Alias thickening**: Workflows that delegate to skills still need Quantis-specific process (state updates, banners, commit protocols) — pure pass-through isn't enough.
3. **Reference sweep is large**: GSD→Quantis rebranding touched 12+ files across skills, templates, workflows. MANIFEST makes future sweeps traceable.
4. **Testing markdown methodology**: Traditional test suites don't apply. Verification is inline (grep sweeps, cross-ref checks, file existence).
