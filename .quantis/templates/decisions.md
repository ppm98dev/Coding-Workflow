# DECISIONS.md — Architecture Decision Records

> **Purpose**: Log significant technical decisions and their rationale.

## Template

```markdown
### D-{NNN}: {Decision Title}
**Decision:** {what was decided}
**Rationale:** {why}
```

**ID allocation:** `NNN` is the next integer after the highest existing `D-` ID across `.quantis/DECISIONS.md`, zero-padded to 3 digits (e.g. `D-001`, `D-042`). Optional fields — add only when they carry weight: `**Date:** YYYY-MM-DD`, `**Status:** Proposed | Accepted | Superseded`, `**Alternatives:** {options considered}`.

This is the canonical decision format. All writers (`/discuss-phase`, `/stress-test`, context-health-monitor) use it.
